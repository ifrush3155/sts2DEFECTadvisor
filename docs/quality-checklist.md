# 质量检查文档

## 1. 检查目标

本文件用于定义《杀戮尖塔 2》机器人游玩辅助的质量检查标准。检查重点包括推荐数据正确性、页面识别准确性、卡牌识别稳定性、UI 可用性和性能表现。

## 2. 推荐数据质量

### 2.1 数据结构

- [ ] 推荐数据文件可以被正常加载。
- [ ] 数据文件包含版本号。
- [ ] 每个类型都有唯一名称。
- [ ] 每个类型都有非空卡牌列表。
- [ ] 每张卡牌都有名称、类型、位次、总数和推荐指数。

### 2.2 排序规则

- [ ] 同类型卡牌顺序与参考图从左到右顺序一致。
- [ ] `rank` 从 1 开始。
- [ ] 同一类型内 `rank` 连续且不跳号。
- [ ] `total` 等于该类型卡牌总数。
- [ ] `recommendIndex` 与 `rank/total` 一致。

### 2.3 更新规则

- [ ] 每次替换参考图后同步更新数据版本。
- [ ] 每次人工修正卡牌名称后记录变更说明。
- [ ] 删除卡牌时同步重算该类型下所有推荐指数。
- [ ] 新增卡牌时同步更新该类型 `total`。

## 3. 页面识别质量

### 3.1 卡牌选择页面

- [ ] 能识别卡牌奖励页面。
- [ ] 能识别其他可能出现可选卡牌的页面。
- [ ] 所有可选卡牌都有对应检测框。
- [ ] 页面离开后推荐标签自动隐藏。
- [ ] 非选牌页面不显示选牌推荐。

### 3.2 卡牌库页面

- [ ] 能识别当前卡牌库页面。
- [ ] 能识别当前可见卡牌。
- [ ] 重复卡牌按实际数量统计。
- [ ] 翻页或滚动后能刷新统计结果。
- [ ] 未识别卡牌进入「未识别」计数，不强行归类。

## 4. 卡牌识别质量

- [ ] 常见分辨率下识别结果稳定。
- [ ] 窗口缩放后识别区域仍然正确。
- [ ] 卡牌名称识别与模板匹配结果一致时优先采用高置信度结果。
- [ ] OCR 结果与模板结果冲突时记录日志。
- [ ] 置信度低于阈值时不直接展示确定结论。

建议初始阈值：

| 项目 | 建议阈值 |
| --- | --- |
| 页面识别置信度 | 0.85 |
| 卡牌模板匹配置信度 | 0.90 |
| OCR 名称置信度 | 0.80 |

## 5. UI 质量

- [ ] 标签不遮挡卡牌名称。
- [ ] 标签不遮挡卡牌费用。
- [ ] 标签不遮挡卡牌描述。
- [ ] 标签不遮挡确认、跳过、关闭等关键按钮。
- [ ] 推荐指数完整显示，不被截断。
- [ ] 低置信度状态有明确视觉区分。
- [ ] 无推荐数据状态有明确提示。
- [ ] 卡牌库统计面板可以折叠。
- [ ] 叠加层不抢占鼠标焦点。

## 6. 性能质量

- [ ] 卡牌选择页面推荐延迟小于 500 ms。
- [ ] 卡牌库统计刷新小于 1 s。
- [ ] 截图与识别不会造成明显游戏卡顿。
- [ ] 后台运行 30 分钟内内存增长稳定。
- [ ] 游戏窗口最小化或丢失时停止高频截图。

## 7. 兼容性质量

- [ ] 支持窗口模式。
- [ ] 支持无边框窗口模式。
- [ ] 支持至少一种主流分辨率，例如 1920x1080。
- [ ] 游戏窗口移动后叠加层位置同步更新。
- [ ] 游戏窗口关闭后工具不会崩溃。

## 8. 发布准入

MVP 发布前必须满足：

- [ ] 推荐数据校验全部通过。
- [ ] 卡牌选择页面核心流程无阻断问题。
- [ ] 卡牌库统计核心流程无阻断问题。
- [ ] 错误归类率低于 1%。
- [ ] 高优先级 UI 遮挡问题为 0。
- [ ] 崩溃级问题为 0。
- [ ] 有一组可复现的截图样本用于回归测试。
