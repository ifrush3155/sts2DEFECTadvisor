# 系统架构文档

## 1. 总体架构

系统由截图、识别、推荐数据、聚合和叠加展示 5 个核心部分组成。

```text
游戏窗口
  -> 屏幕捕获模块
  -> 页面识别模块
  -> 卡牌识别模块
  -> 结果聚合服务
  -> 叠加层 UI

参考图
  -> 推荐数据构建器
  -> 推荐数据服务
  -> 结果聚合服务
```

## 2. 模块职责

### 2.1 屏幕捕获模块

职责：

- 绑定目标游戏窗口。
- 获取窗口位置和尺寸。
- 周期性截取窗口画面。
- 在窗口最小化、关闭或失焦时输出状态。

输出：

```json
{
  "frameId": "2026-06-27T15:30:00.123",
  "windowBounds": { "x": 100, "y": 100, "width": 1920, "height": 1080 },
  "image": "<frame>"
}
```

### 2.2 页面识别模块

职责：

- 判断当前页面类型。
- 避免在非目标页面显示推荐。

页面类型：

```text
CARD_REWARD
CARD_LIBRARY
GAMEPLAY
UNKNOWN
WINDOW_LOST
```

### 2.3 卡牌识别模块

职责：

- 检测画面中的卡牌区域。
- 识别卡牌名称。
- 输出识别置信度。

MVP 建议优先使用模板匹配，OCR 作为辅助。模板匹配适合稳定 UI，OCR 用于补充卡牌名称识别。

输出：

```json
{
  "pageType": "CARD_REWARD",
  "cards": [
    {
      "name": "Card A",
      "confidence": 0.94,
      "bounds": { "x": 420, "y": 220, "width": 180, "height": 260 }
    }
  ]
}
```

### 2.4 推荐数据服务

职责：

- 加载本地推荐数据。
- 校验数据完整性。
- 根据卡牌名称查询类型和推荐指数。
- 提供版本信息和数据校验结果。

推荐数据结构：

```json
{
  "version": "2026-06-27-manual-001",
  "source": {
    "type": "manual",
    "description": "由用户提供的参考图人工标注生成"
  },
  "types": [
    {
      "name": "攻击接口",
      "cards": [
        {
          "name": "Card A",
          "rank": 1,
          "total": 8,
          "recommendIndex": "1/8"
        }
      ]
    }
  ]
}
```

### 2.5 结果聚合服务

职责：

- 合并卡牌识别结果和推荐数据。
- 标记低置信度、无数据、未识别等状态。
- 生成 UI 可直接消费的数据模型。
- 在卡牌库页面生成类型统计。

卡牌库统计结构：

```json
{
  "totalCards": 18,
  "unknownCount": 1,
  "byType": {
    "攻击接口": {
      "count": 4,
      "recommendIndexes": ["1/8", "2/8", "5/8", "5/8"]
    }
  }
}
```

### 2.6 叠加层 UI

职责：

- 跟随游戏窗口位置显示。
- 渲染卡牌标签。
- 渲染卡牌库统计面板。
- 响应设置变更。
- 避免抢占游戏输入焦点。

## 3. 推荐技术栈

MVP 推荐使用：

- Python：主开发语言。
- OpenCV：图像处理、模板匹配。
- PySide 或 PyQt：叠加层窗口和设置界面。
- JSON：推荐数据配置。
- SQLite：可选，用于保存识别日志和历史结果。

选择原因：

- Python 图像识别生态成熟。
- OpenCV 能快速验证截图和模板匹配流程。
- PySide / PyQt 适合构建本地叠加层工具。
- JSON 便于人工维护推荐数据。

## 4. 数据流

### 4.1 卡牌选择页面

```text
截图
-> 页面识别为 CARD_REWARD
-> 检测可选卡牌
-> 识别卡牌名称
-> 查询推荐数据
-> 生成卡牌标签
-> 叠加显示
```

### 4.2 卡牌库页面

```text
截图
-> 页面识别为 CARD_LIBRARY
-> 检测所有可见卡牌
-> 识别卡牌名称
-> 查询推荐数据
-> 统计类型数量和推荐指数数组
-> 显示卡牌标签和统计面板
```

## 6. 迭代路线

### 6.1 第一阶段：可运行原型

- 完成窗口绑定。
- 完成截图。
- 使用固定截图样本验证卡牌区域检测。
- 人工维护推荐数据 JSON。
- 在测试窗口上显示叠加标签。

### 6.2 第二阶段：MVP

- 接入实时游戏窗口。
- 完成卡牌选择页面推荐。
- 完成卡牌库统计面板。
- 增加识别日志。
- 完成基础设置界面。

### 6.3 第三阶段：增强能力

- 支持参考图辅助生成推荐数据。
- 支持更多分辨率和窗口模式。
- 增加回归测试样本集。
- 优化低置信度处理。

## 7. 主要风险

- 游戏 UI 变化导致模板匹配失效。
- 参考图未结构化时推荐数据容易录入错误。
- OCR 对卡牌字体和语言环境敏感。
- 叠加层可能在部分窗口模式下无法正确置顶。
- 高频截图可能影响游戏性能。

对应缓解：

- 保留截图样本用于回归测试。
- 推荐数据加载时做严格校验。
- 模板匹配和 OCR 结果互相校验。
- 提供截图频率和置信度阈值设置。
- 窗口丢失或非目标页面时降低识别频率。
