# UI 设计文档

## 1. 设计目标

辅助工具的 UI 应作为游戏画面的轻量叠加层出现。它的目标是提供决策参考，而不是替代玩家判断。

设计原则：

- 信息足够清晰，但不抢占画面中心。
- 默认不遮挡卡牌核心内容。
- 低置信度和无数据状态必须可被一眼识别。
- 卡牌库统计面板适合快速扫读。

## 2. 整体布局

UI 分为 3 类区域：

- 卡牌标签：显示在单张卡牌附近。
- 卡牌库统计面板：显示在卡牌库页面右侧。
- 状态提示：显示窗口绑定、识别暂停、数据异常等全局状态。

## 3. 卡牌选择页面

### 3.1 普通标签

普通识别成功时，每张卡牌下方显示一个小标签：

```text
攻击接口
推荐 2/8
```

标签内容：

- 第一行：卡牌类型。
- 第二行：推荐指数。

### 3.2 低置信度标签

当识别置信度低于阈值但仍有候选结果时，显示：

```text
可能是 攻击接口
推荐 2/8
置信度低
```

低置信度标签需要使用更克制的颜色，并避免与确定结果混淆。

### 3.3 无数据标签

识别到卡牌但推荐表中没有数据时，显示：

```text
无推荐数据
```

该状态不应显示虚假的类型或推荐指数。

## 4. 标签位置规则

优先级从高到低：

1. 显示在卡牌下方。
2. 下方空间不足时显示在卡牌右侧。
3. 靠近屏幕边缘时自动向内偏移。
4. 多个标签重叠时垂直错开。

标签不得遮挡：

- 卡牌名称。
- 卡牌费用。
- 卡牌描述。
- 确认、跳过、关闭等操作按钮。

## 5. 卡牌库页面

卡牌库页面右侧显示统计面板。面板默认展开，支持折叠。

示例：

```text
当前卡牌库
总数 18    未识别 1

攻击接口  4
[1/8, 2/8, 5/8, 5/8]

防御接口  2
[1/6, 4/6]

零费接口  3
[1/5, 3/5, 4/5]
```

### 5.1 面板字段

- 总数：当前识别到的卡牌数量。
- 未识别：未能匹配推荐数据的卡牌数量。
- 类型名称：来自推荐数据配置。
- 类型数量：当前卡牌库中该类型卡牌数量。
- 推荐指数数组：该类型下所有卡牌的推荐指数。

### 5.2 排序规则

默认排序建议：

1. 按类型在推荐配置中的顺序展示。
2. 同类型内按推荐指数 `rank` 从小到大展示。
3. 相同推荐指数保留重复项。

## 6. 设置界面

设置界面用于调整识别和显示行为。

建议设置项：

- 选择游戏窗口。
- 导入推荐数据。
- 叠加层透明度。
- 截图频率。
- 页面识别阈值。
- 卡牌识别阈值。
- 是否显示低置信度结果。
- 是否显示无推荐数据标签。
- 查看识别日志。

## 7. 状态提示

### 7.1 未绑定窗口

```text
请选择游戏窗口
```

### 7.2 窗口丢失

```text
游戏窗口不可用
```

### 7.3 推荐数据异常

```text
推荐数据加载失败
```

### 7.4 识别暂停

```text
识别已暂停
```

状态提示应出现在画面边缘，不覆盖游戏主操作区域。

## 8. 视觉风格

- 使用半透明深色背景，保证在亮色和暗色游戏场景中都可读。
- 推荐标签使用小尺寸文本，避免形成视觉噪声。
- 确定结果、低置信度、无数据状态使用不同颜色区分。
- 统计面板保持紧凑，避免大面积遮挡卡牌库。
