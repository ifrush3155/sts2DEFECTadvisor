# 杀戮尖塔 2 机器人游玩辅助：项目需求文档

## 1. 项目背景

本项目用于开发一款本地运行的《杀戮尖塔 2》机器人游玩辅助工具。工具不直接接管玩家操作，MVP 阶段聚焦于「识别当前页面」「展示卡牌推荐」「统计当前卡牌库」三类能力，帮助玩家在选牌和查看卡牌库时快速理解当前卡牌价值与构筑结构。

推荐信息来自用户提供的参考图。参考图按照卡牌类型分组，每组内卡牌从左到右排列。卡牌的推荐指数由其在所属类型中的位次和该类型总数共同决定。

## 2. 项目目标

- 在进入卡牌选择页面时，自动识别当前可选择卡牌，并在游戏画面上叠加推荐信息。
- 在打开当前卡牌库时，识别当前所有卡牌，展示每张卡牌的推荐信息。
- 按卡牌类型统计数量，并展示各类型下所有卡牌的推荐指数数组。
- 支持通过本地配置维护参考图解析后的推荐数据，方便后续迭代和校正。

## 3. 用户场景

### 3.1 卡牌选择

玩家进入卡牌奖励或卡牌选择页面后，辅助工具自动识别页面中的可选卡牌，并在每张卡牌附近显示：

- 类型：例如攻击接口、防御接口、零费接口。
- 推荐指数：格式为 `a/b`。

其中：

- `a` 表示该卡牌在所属类型中从左到右的位次。
- `b` 表示该类型在参考图中的卡牌总数。

### 3.2 卡牌库查看

玩家打开当前卡牌库时，辅助工具自动识别可见卡牌，并展示：

- 每张卡牌的类型。
- 每张卡牌的推荐指数。
- 每种类型的卡牌数量。
- 每种类型下所有卡牌的推荐指数数组。

示例：

```json
{
  "攻击接口": {
    "count": 4,
    "recommendIndexes": ["1/8", "2/8", "5/8", "5/8"]
  },
  "防御接口": {
    "count": 2,
    "recommendIndexes": ["1/6", "4/6"]
  }
}
```

## 4. 功能范围

### 4.1 MVP 范围

- 选择并绑定游戏窗口。
- 周期性截取游戏窗口画面。
- 判断当前页面是否为卡牌选择页面或卡牌库页面。
- 识别当前页面中的卡牌区域。
- 根据卡牌名称或模板匹配结果查询推荐数据。
- 在游戏窗口上方显示透明叠加层。
- 在卡牌选择页面显示卡牌推荐标签。
- 在卡牌库页面显示推荐标签和统计面板。
- 记录识别失败、低置信度、无推荐数据等异常情况。

### 4.2 暂不包含

- 自动点击卡牌。
- 自动选牌。
- 自动出牌。
- 自动规划完整爬塔路线。
- 在线同步推荐数据。
- 基于胜率模型的动态推荐。

这些能力可以作为后续阶段扩展，但不应进入 MVP，避免第一版范围过大。

## 5. 推荐数据规则

推荐数据由参考图转换为结构化配置。每个类型包含一组卡牌，卡牌在数组中的顺序必须与参考图从左到右的顺序一致。

推荐指数计算规则：

```text
recommendIndex = rank / total
```

- `rank` 从 1 开始。
- `total` 等于该类型下卡牌总数。
- 同一类型内 `rank` 不允许重复，除非后续明确支持同图多版本或同名异卡。
- 卡牌库中出现多张相同卡牌时，统计时应按实际拥有数量重复计入。

## 6. 数据输入与输出

### 6.1 输入

- 游戏窗口截图。
- 参考图解析后的推荐数据 JSON。
- 用户设置项，例如识别频率、透明度、置信度阈值。

### 6.2 输出

卡牌选择页面输出：

```json
[
  {
    "cardName": "Card A",
    "type": "攻击接口",
    "recommendIndex": "2/8",
    "confidence": 0.94,
    "bounds": { "x": 420, "y": 220, "width": 180, "height": 260 }
  }
]
```

卡牌库页面输出：

```json
{
  "cards": [
    {
      "cardName": "Card A",
      "type": "攻击接口",
      "recommendIndex": "2/8",
      "confidence": 0.94
    }
  ],
  "summary": {
    "攻击接口": {
      "count": 1,
      "recommendIndexes": ["2/8"]
    }
  }
}
```

## 7. 异常处理

- 未识别到游戏窗口：显示窗口绑定提示，不展示推荐信息。
- 页面类型未知：隐藏推荐叠加层，仅记录日志。
- 卡牌识别置信度低：显示「可能是」标签，并标记低置信度。
- 识别到卡牌但无推荐数据：显示「无推荐数据」。
- 推荐数据文件损坏：停止推荐展示，并提示重新导入。

## 8. 验收标准

- 进入卡牌选择页面后，500 ms 内出现推荐标签。
- 离开卡牌选择页面后，推荐标签自动消失。
- 打开卡牌库后，1 s 内生成统计面板。
- 重复卡牌按实际数量统计。
- 未识别卡牌不应被错误归入某个类型。
- 推荐指数与参考图顺序保持一致。
- UI 不遮挡卡牌名称、费用、描述和关键按钮。
